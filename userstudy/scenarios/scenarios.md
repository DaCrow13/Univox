# Experimental Scenarios

This document lists the scenarios used for the user study. Each participant was assigned a subset of these scenarios.

## 1. RecSys Fundamentals (MRI)
- **Subject**: Methods for Information Retrieval (MRI)
- **Role**: You are an MRI student studying Recommender Systems.
- **Context**: You need to clarify the fundamental problem these systems solve.
- **Goal**:
    1. Ask: "Secondo le slide introduttive del corso MRI, quale problema principale risolvono i sistemi di Information Filtering e Recommender Systems?"
    2. Verify if the answer mentions **Information Overload**.

## 2. Content-Based Filtering (MRI)
- **Subject**: MRI
- **Role**: You are diving into Content-Based recommendation methods.
- **Context**: You want to understand how items are represented in this approach.
- **Goal**:
    1. Ask: "In un sistema Content-Based, come viene tipicamente rappresentato il profilo di un item?"
    2. Verify if the answer refers to **features** or **attributes** (e.g., keywords, metadata).

## 3. Evaluation Metrics (MRI)
- **Subject**: MRI
- **Role**: You need to evaluate the performance of a recommender system.
- **Context**: You need to know which metrics to use to measure accuracy.
- **Goal**:
    1. Ask: "Quali sono le principali metriche di errore utilizzate per valutare l'accuratezza di predizione nei Recommender Systems?"
    2. Verify if the answer includes **MAE** (Mean Absolute Error) or **RMSE** (Root Mean Squared Error).

## 4. Semantic Web Vision (SIIA)
- **Subject**: Semantics in Intelligent Information Access (SIIA)
- **Role**: You are an SIIA student starting to study the Semantic Web.
- **Context**: You want to understand Tim Berners-Lee's original vision.
- **Goal**:
    1. Ask: "Qual è l'idea centrale del Semantic Web secondo la visione di Tim Berners-Lee?"
    2. Verify if the answer speaks about extending the Web to make data **machine-readable and processable**.

## 5. RDF Triples (SIIA)
- **Subject**: SIIA
- **Role**: You are studying the RDF data model.
- **Context**: You need a concrete example of how information is structured.
- **Goal**:
    1. Ask: "Spiegami la struttura di una tripla RDF e fammi un esempio."
    2. Verify if the response describes the **Subject-Predicate-Object** structure.

## 6. Python Typing (LP)
- **Subject**: Programming Languages (LP)
- **Role**: You are a Programming Languages student interested in Python.
- **Context**: You want to understand how Python handles types compared to languages like Java.
- **Goal**:
    1. Ask: "Che tipo di tipizzazione utilizza Python e in cosa differisce dalla tipizzazione statica?"
    2. Verify if the answer mentions **dynamic typing** or **duck typing**.

## 7. Java Interfaces vs Abstract Classes (LP)
- **Subject**: LP
- **Role**: You are reviewing OOP concepts in Java.
- **Context**: You need to clarify when to use an interface or an abstract class.
- **Goal**:
    1. Ask: "Qual è la differenza principale tra una classe astratta e un'interfaccia in Java (pre-Java 8)?"
    2. Verify if the response explains that a class can implement **multiple interfaces** but extend only **one class**.

## 8. Haskell Functions (LP)
- **Subject**: LP
- **Role**: You are curious about functional programming in Haskell.
- **Context**: You heard about higher-order functions.
- **Goal**:
    1. Ask: "Cos'è una funzione di ordine superiore (Higher-Order Function) in Haskell?"
    2. Verify if the answer says it's a function that takes **other functions as arguments** or returns a function.
